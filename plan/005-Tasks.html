Tasks levels:

  1. Basic tasks (critical path, required)

    • each participant have to complete them
    • required to do tasks from next Sprint
    • should be possible to do in 5-10 minutes
    • participants should do them as first

  2. Additional / Advanced tasks (nice to have features)

    • additional tasks, within the scope of Sprint
    • some additional features, more advanced usage
    • should require something more then copy-paste from materials

  3. Extra tasks (wow effect)

    • tasks beyond the scope of Sprint (or even workshop)
    • can utilize features from next Sprint (without properly introducing them)
    • can significantly extend functionality of the app
    • for very good, active participants

Prepare tasks

In each Sprint you need to define what participants should do - what new features the App will have.

On each and every Devmeeting some participants will be able to do more/less than others - be prepared!

Tasks should be focused on app features, but should also always teach participants something about the Devmeeting topic.
