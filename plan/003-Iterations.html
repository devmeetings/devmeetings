Couple of rules to have in mind when preparing iterations:

  1. Allow people to make mistakes
    • Don't teach them everything right from the start,
    • Let them make something in a wrong way, to propose better solution in next iteration.
  2. Start small
    • First think about minimal app the is tangible (renders something) and define minimal set of language / framework features to achieve that,
    • Keep code at minimum - it's better to learn basic stuff correctly, don't re-write documentation.
  3. Define critical path
    • What's minimal increment to be done in a step to be able to attempt next iteration.

Split app into iterations

So you already know that participants will write an app.

We believe that the app should be developed Agile (Scrum-like) way.

App development should be split into iterations, each iteration should result in Minimum Viable Product (MVP) and should also show some new features of language / framework.
